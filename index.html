中文

What is NJES?

NJES means Not Just Embedded Scripting language. This is a efficient, lightweight, embeddable scripting language. It designed supports procedural programming, object-oriented programming, functional programming, and data description.

NJES combines simple procedural syntax with powerful data description constructs based on associative arrays and extensible semantics. NJES is dynamically script, runs by interpreting bytecode with a register-based virtual machine, and has automatic memory management with incremental garbage collection, making it ideal for configuration , Scripting, and rapid prototyping.

NJES only contains the basic functions and interpreters, all functional functions are provided by the operating environment.

Why create NJES?

Because the existing embedable script language is not free to define the extension library, or explain the implementation is not convenient enough. Even if suitable as a quick script, but also because it is not like-C syntax and become a bit awkward. So we created NJES, in order to reduce the cost of learning as a quick script. And still maintain freedom and interoperability.
